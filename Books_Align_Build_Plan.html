1 Project Overview

Books Align is a Nepal-based accounting firm specialising in NDIS (National Disability Insurance Scheme) financial services for Australian providers. This document covers the full build plan for their professional website — from the already-complete React frontend through to a production-ready Express/MongoDB backend with Google OAuth, booking management, and an admin dashboard.

✓ Custom Google Meet booking flow ✓ Separate /admin dashboard route ✓ Render deploy → Hostinger production

5 Google Meet Booking Flow

The booking system uses Google OAuth to authenticate users, Google Calendar API to check availability and create events, and automatically generates a Google Meet link for each confirmed booking.

Required Google API Scopes

https://www.googleapis.com/auth/calendar.events — create and manage calendar events
profile email openid — standard OAuth identity scopes

Step-by-Step Flow

StepActorAction
1UserClicks "Book a Time" on the Contact page
2FrontendChecks AuthContext — if not logged in, redirects to GET /api/auth/google
3BackendPassport redirects to Google OAuth consent screen with calendar.events scope
4GoogleUser grants permission; Google redirects to /api/auth/google/callback
5BackendPassport exchanges code for tokens; user saved/updated in MongoDB; JWT issued in HttpOnly cookie
6FrontendAuthContext detects session; booking widget unlocks; slot picker is displayed
7FrontendCalls GET /api/booking/slots?date=YYYY-MM-DD to fetch available 30-min slots
8BackendQueries Google Calendar free/busy API for the admin's calendar; returns open slots
9UserSelects a slot and clicks "Confirm Booking"
10FrontendPosts POST /api/booking with selected slot
11BackendDouble-checks slot is still free (race condition guard)
12BackendCreates Google Calendar event with conferenceDataVersion: 1 → generates Meet link
13BackendSaves booking to MongoDB; sends confirmation emails to user and admin via Nodemailer
14FrontendShows confirmation card with Google Meet link and datetime

8 Security Implementation

LayerPackageWhat It Does
HTTP HeadershelmetSets Content-Security-Policy, X-Frame-Options, etc.
Rate Limitingexpress-rate-limit100 req/15 min globally; 5 req/15 min on /api/contact; 10 req/15 min on /api/auth
CORScorsWhitelist frontend origin only; credentials: true
NoSQL Injectionexpress-mongo-sanitizeStrips $ and . from req.body to prevent MongoDB operator injection
XSSxss-cleanSanitises req.body, req.query, req.params
HTTP PollutionhppPrevents duplicate query params from bypassing validation
Auth TokensjsonwebtokenShort-lived JWT (1h) in HttpOnly, Secure, SameSite=Strict cookie
SpamGoogle reCAPTCHA v3Verified server-side on POST /api/contact
Input ValidationZodStrict schemas on all POST/PATCH bodies
Error HandlingCustom middlewareNever leaks stack traces in production; structured JSON errors

10 Admin Dashboard (/admin)

A separate, protected React route. Admin logs in with email + password (not Google OAuth — keeps admin access independent of Google accounts). JWT stored in HttpOnly cookie identical to user auth.

Dashboard Layout

ViewFeatures
Login /admin/loginEmail + password form, matches the Books Align design system
Dashboard /adminStats cards (total contacts, bookings this week, pending), recent 5 contacts, next 3 bookings
Contacts /admin/contactsTable: name, email, service, date, status badge, mark-read button, delete
Bookings /admin/bookingsTable: user name, email, date/time, Meet link, status badge, cancel button
